# Grimoire Vault — гайд пользователя

> Личная база знаний — тринадцать комнат, один пароль, мульти-устройство, Telegram-бот.

Этот документ — для того, кто уже зашёл в свой vault и хочет понять, что
здесь делать.  Если ты разработчик — открой [`DEVELOPER.md`](./DEVELOPER.md).
Если только смотришь и не уверен, нужен ли тебе этот инструмент —
[`GETTING-STARTED.md`](./GETTING-STARTED.md) даст обзор за 5 минут.

---

## 1. Что это

Личный «второй мозг» в браузере + Telegram.  Ты:

1. Кидаешь в бота ссылку, заметку или фото — оно автоматически попадает
   в нужную категорию с подтянутым заголовком.
2. На сайте видишь это в Inbox-очереди и за два клика подтверждаешь
   или перемещаешь в правильное место.
3. Когда нужна старая идея — поиск находит её по словам или по смыслу.

Дублирующая ссылка — не сохранится.  Утерянное устройство —
ничего не теряешь, всё в облаке.  Решил уйти на другой хостинг — экспорт
и импорт за минуту.

## 3. Как добавлять записи

### 3.1 Через Telegram-бота — самый быстрый путь

1. Найди в Telegram бота `@TheBaseofKnowladge_bot`.
2. Открой `/settings → Telegram`, нажми «Issue link code» → получишь
   код вида `482-913`.
3. В чате с ботом отправь `/link 482-913`.
4. Готово.  Теперь любая ссылка / текст / фото, отправленные боту,
   автоматически сохраняются:
   - YouTube-ссылка → раздел «YouTube» с заголовком и превью
   - Любая другая ссылка → «Web Resources» с подтянутым `og:title` и описанием
   - Текст без ссылки → «Ideas»
   - Фото → «Images»

Бот отвечает в чате, в какую категорию положил.  Если уже сохранял эту
ссылку — ответ будет «🔁 Уже сохранено в …».

### 3.2 Через сайт — кнопка «Add Entry» в любой категории

Открываешь категорию, жмёшь «Add Entry», заполняешь поля.

В Web Resources, если вставишь URL, через пол-секунды подтянется
заголовок, описание и thumbnail — править ничего не надо, только нажать
«Сохранить».

### 3.3 Через ⌘K (Cmd+K на Mac, Ctrl+K на Windows) — для опытных

Открой палитру откуда угодно.  Вставь ссылку — палитра автоматически
определит категорию (YouTube → YouTube, GitHub → Web, Figma → Designs)
и предложит «Сохранить в …».  ↑↓ переключает категорию, ↵ — сохраняет.

## 4. Inbox — очередь к разбору

Бот валит всё подряд в Inbox с пометкой «не разобрано».  В шапке сайта
рядом с «Inbox» горит золотой счётчик — сколько ждёт.

В разделе `/inbox`:

- **Filed** — подтвердить категорию (бот угадал верно), запись уйдёт из очереди.
- **Переместить** — дропдаун из 13 категорий, выбираешь правильную.
- **Удалить** — мусор.

Всё работает массово: ткни чекбоксы у нескольких записей → внизу
появится тулбар → одним кликом обработаешь всю пачку.

**Цель:** держать счётчик на нуле.  Если нарастает — значит, ритуал
триажа не работает, а не приложение.

## 5. Поиск — три режима

### 5.1 Точное · слова

Postgres FTS с русской морфологией.  Ищет по точным словам или их
формам.  «программа» найдёт и «программирование», и «программу».

### 5.2 Гибрид · RRF (по умолчанию)

Объединяет точный поиск со смысловым через Reciprocal Rank Fusion.
Лучший выбор «не задумываясь».  Находит и редкое имя, и парафраз.

### 5.3 По смыслу · AI

Семантический поиск через локальные эмбеддинги.  Ищет по идее, а не по
буквам.  «как настроить телеграм-бот» найдёт записи, где этих слов нет,
но смысл совпадает.

> **Первый раз** при выборе семантического или гибридного режима в
> браузер скачивается языковая модель ~30 MB.  Дальше работает офлайн и
> мгновенно.

### 5.4 Bulk-операции в результатах поиска

Shift+Click по результату → добавляешь в выделение.  Внизу появляется
тулбар: добавить тег / перенести / удалить.  Полезно для разбора:
«найди все про дизайн → пометь тегом design-2026».

## 7. Kanban-доска

`/kanban` — три колонки: Backlog, Doing, Done.  Drag-and-drop работает
с мышью, тачем и клавиатурой.  Карточки синхронизируются realtime между
устройствами.

Это **личная** доска — не делится с членами shared vault'а (см. раздел 9).

---

## 8. Credentials — зашифрованный vault

`/category/credentials` — отдельный раздел для логинов / паролей /
заметок.  Шифрование AES-GCM-256, ключ выводится из мастер-пароля
через PBKDF2 (600k итераций).

**Мастер-пароль никогда не покидает браузер.**  Сервер видит только
зашифрованные блобы.  Если ты забыл мастер-пароль — данные потеряны
безвозвратно.  Никакой «password reset» здесь не поможет.

При первом заходе придумываешь мастер-пароль, потом он спрашивается
один раз за сессию (хранится в `sessionStorage`, обнуляется при
закрытии вкладки).

Credentials остаются **личными** даже в shared vault'е — нельзя
безопасно расшарить, потому что shared vault не подразумевает shared
master-password.

## 9. Shared vaults — для семьи / команды

В `/settings → Vaults`:

1. **Создай vault** с именем (например, «Family · Recipes» или
   «Travel-2026»).
2. **Сгенерируй invite-ссылку.**  Срок действия — 7 дней.
3. **Отправь её другому человеку.**  Он откроет её в браузере, войдёт
   через свой email/пароль, и автоматически попадёт в vault как editor.

В шапке сайта появится **VaultPicker** — переключатель между Personal
и shared vault'ами.  Выбираешь «Family» — все записи, которые ты
добавляешь, попадают в этот vault и видны всем участникам.  Переключаешь
на «Personal» — снова приватный режим.

**Роли:**
- `owner` — создатель vault'а; может приглашать, кикать, удалять vault.
- `editor` — может CRUD'ить записи внутри.

**Что НЕ шарится:**
- Kanban-доска (личная)
- Credentials (зашифрованы под твой мастер-пароль)
- Telegram-бот (всё, что ты ему кинешь, идёт в твой Personal)

> Если хочешь, чтобы бот клал записи в shared vault — сейчас не умеет.
> Перемести вручную через «Переместить» в inbox.

## 10. Уведомления — Web Push

`/settings → Уведомления → Включить`.

Браузер один раз спросит разрешение, дальше работает в фоне:

- На **Android Chrome / desktop Chrome / Firefox** — нативные системные пуши.
- На **iOS Safari (16.4+)** — нужно сначала **установить PWA на главный
  экран**: открой сайт в Safari → Share → Add to Home Screen.  После
  этого открой PWA, зайди в `/settings`, разреши push.

Что приходит в виде пуша:

- Новая запись от бота → пуш с категорией и заголовком.
- (В будущем — действия в shared vault'ах.)

«Послать тест» в Settings — кнопка для проверки round-trip'а без
ожидания реального события.

## 11. Бэкап и миграция

В `/settings → Export Vault`:

- **JSON-бэкап** — всё содержимое БД (entries, kanban, credentials в
  ciphertext), без R2-файлов.  Лёгкий — обычно несколько MB.
- **Full export · ZIP** — то же самое + все binaries из R2 (картинки,
  файлы).  Тяжёлый — может быть сотни MB.

`/settings → Import Vault` — обратная операция.  Любой JSON-бэкап
импортируется, дубли автоматически пропускаются.  Полезно для:
- Миграции на другой аккаунт / другой деплой.
- Восстановления после случайного удаления.
- Слияния двух vault'ов.

**Совет:** раз в месяц делай Full export → закидывай в iCloud / Google
Drive.  Это твоя страховка от того, что Supabase / Vercel когда-нибудь
испарятся.

## 12. Установка PWA на телефон

### iOS

1. Открой `https://grimoire-vault.vercel.app` в **Safari** (не Chrome!).
2. Кнопка Share (квадрат со стрелкой).
3. «Add to Home Screen».

3. То же самое.

кеширует статику и последние просмотренные категории.

## 14. Частые вопросы

**Я добавил ссылку через бота, она не появилась в категории.**
→ Проверь, что ты действительно привязал чат через `/link <код>`.
   Если не привязан, бот ответит «Сначала привяжи свой аккаунт».

**Embedding coverage в Settings показывает <100%.**
→ Это нормально для записей, импортированных через бота — у них
   embedding=null.  Нажми `Settings → Поисковые embeddings → Запустить
   бэкфилл` один раз.  Дальше всё новое будет проиндексировано
   автоматически.

**Семантический поиск ничего не находит.**
→ Coverage 0%? Запусти бэкфилл.  Иначе попробуй другие формулировки
   запроса — на русском e5-small работает хуже, чем на английском.

**Я забыл мастер-пароль для Credentials.**
→ Ничего нельзя сделать.  Зайди в `/category/credentials`, удали все
   записи, поставь новый мастер-пароль и заведи credentials заново.
   Старые блобы — необратимый шифротекст.

**Хочу добавить свою категорию (14-ю).**
→ Не предусмотрено.  Используй теги.  Если категория действительно
   нужна — это разработческая задача, см. `DEVELOPER.md → Adding a category`.

**Можно ли сделать vault'у пароль на вход?**
→ Нет.  Доступ к vault'у определяется членством (`vault_members`
   table).  Для приватности — оставайся в Personal-mode.

## 15. Куда смотреть, если что-то сломалось

1. **Что-то лагает / не сохраняется** → открой DevTools → Network → найди
   красный запрос → скопируй заголовок `X-Request-Id`.  Это твой
   референс для разработчика.
2. **Бот молчит** → `/admin/health` (доступно только владельцу деплоя)
   покажет статус Telegram webhook'а.
3. **Realtime не обновляет** → перезагрузи страницу.  Supabase
   Realtime иногда теряет соединение, но восстанавливается при
   следующем запросе.
4. **Совсем сломалось** → `Settings → Full export` (сохрани!) → можно
   потом пересоздать vault, что-то делать дальше.

## 16. Минимальный ритуал для долгожителя

- 🌅 **Утром** — `/inbox`, разобрать всё до нуля (2-5 минут).
- 📥 **В течение дня** — кидай ссылки боту, не пытайся «потом
  запомнить».
- 🔍 **Когда ищешь** — сначала vault, потом Google.  Если ищешь то, что
  уже искал в vault'е — добавь как entry.
- 📦 **Раз в месяц** — Full export, сохранить в облако.
- 🔁 **Раз в неделю** — пробежаться по `Settings → Vault stats`, увидеть,
  какие категории дышат, какие мёртвые.

3 недели такого ритуала, и vault станет рефлексом.  Меньше — он умрёт
в анабиозе.

## 17. Сортировка и фильтр по тегам

В шапке списка любой категории — справа от «Все записи · N» — pill-
кнопка с иконкой сортировки.  Клик открывает дропдаун:

| Режим | Что делает |
|---|---|
| Новые | По умолчанию, `created_at DESC` |
| Старые | `created_at ASC` |
| А–Я / Я–А | По названию, RU-aware |
| **По тегам** | По первому тегу алфавитно, без-теговые в конце |

Выбор запоминается **отдельно для каждой категории** в localStorage
(`grimoire:sort:<id>`).  Перезагрузка не сбрасывает.

При выборе «По тегам» появляется **ряд чипов** со всеми тегами в
скоупе и счётчиками.  Клик по тегу — список фильтруется только до
карточек с этим тегом, чип становится золотым.  «Все · N» снимает
фильтр.  Счётчики всегда суммируются с «Все».

## 18. Категория-специфичный UX

### Ideas — плиточная сетка
4 колонки `aspect-square` тайлов вместо плоского списка.  Лампочка
сверху-слева, название, описание (clamp до 3 строк), теги + дата
снизу.  Pinned-идеи — 3 колонки `aspect-[4/3]` с увеличенным шрифтом.

### Prompts — copy-the-prompt
Ховер на карточку → правая верхняя кнопка-копирка кладёт **текст
промпта** в буфер (не URL источника).  Иконка меняется на ✓ на 1.4 с
как подтверждение.  В форме: «Название → Текст промпта → Ссылка →
Модель → Теги».  Селектор модели — стилизованный, не нативный.

### Skills — paste full command
В поле «Ссылка» можно вставить целую shell-команду —
`npx skills add https://github.com/foo/bar --skill x`.  Поле сохранит
строку как есть, но название и описание подтянутся со страницы по
ссылке внутри.  Hover-copy → копирует команду.

### Active Projects — рабочая панель
Категория «Активные проекты» (бывшая Portfolio) — полноценный
проект-трекер.  При создании 3 поля помимо обычных: **Vercel /
прод-ссылка**, **GitHub / репозиторий**, **БД / админ-панель**.
Прод-ссылка автоматически подтягивает название, описание и обложку
через og:meta (с переводом на русский).

На странице записи (`/entry/<id>`) для проектов есть отдельная
панель «Детали проекта»:

- **Быстрые ссылки** — золотые чипы Vercel/Git/БД, открывают в
  новой вкладке
- **ТЗ** — большая textarea, автосейв через 800 мс
- **Дополнительные ссылки** — массив с inline add/edit/remove
- **Доступы и пароли** — массив `{ label, value }` с глаз-toggle и
  copy-кнопкой.  ⚠ Хранятся как plain text (не E2E) — для настоящих
  секретов используй категорию Credentials.

В шапке категории — editable-кнопка «Открыть сайт с работами»
(хранится в localStorage, заполнить когда сайт появится).

### Kanban — кастомные колонки
- В конце ряда колонок — пунктирная плитка «+ Колонка».  Inline-
  инпут с Enter/Escape.
- Заголовок любой колонки на ховере показывает 🖉 (переименовать) и
  для кастомных — × (удалить, только пустые).
- Дефолты `Backlog / Doing / Done` тоже **переименовываемы**, слаги
  при этом фиксированы (на них завязана стилизация).
- **Edit карточки** — карандаш на ховере карточки или двойной клик.
- Drag&drop теперь плавный — никаких «телепортов» между колонками.

## 19. Загрузка картинок — автосжатие

При загрузке любой растровой картинки (JPEG / PNG / BMP / TIFF) она
**автоматически пересжимается** в WebP на клиенте перед заливкой:

- Скейлинг до 2400px по длинной стороне
- Кодирование @ quality 0.82
- Адаптивно: если не влезает в лимит (10 MB для обложек, 5 MB для
  превью видео) — quality снижается шагами до 0.4, потом размеры
  ×0.75 до floor 480px
- Если результат вышел больше оригинала — оставляем оригинал
- WebP / AVIF / GIF / SVG не трогаются

Что увидишь:
- Дроп-зона: «Сжимаю…» → «Загружаю…»
- На карточке: чип с реальным форматом и размером, например
  `WEBP · 412 KB`
- HEIC с iPhone не поддерживается браузером — будет явное
  сообщение «сохрани как JPEG или PNG»

Net эффект: типичная фотка с телефона 5 MB → ~400 KB на R2 без
визуальной деградации.

## 20. Hover-copy на карточках

Для категорий **Skills / Prompts / Ideas / Active Projects / Misc** на
ховере карточки появляется кнопка-копи:

- Skills / Active Projects / Ideas / Misc → копирует `url` (команда

- Prompts → копирует **текст промпта** (description), готовый к
  вставке в Claude / ChatGPT / Gemini

Иконка меняется на галочку на 1.4 секунды после клика — мгновенная
визуальная отдача без alert'ов.

## 21. Auto-translate og:meta

Когда вставляешь URL английского источника (GitHub-репо,
англо-туториал, Vercel-деплой), `/api/extract` подтягивает og:title
и og:description, и они **автоматически переводятся на русский**
через Google Translate gtx (key-less, на лету).

Если контент уже на русском (≥30 % кириллицы) — не трогаем,
оставляем как есть.  Если перевод сетевой упал — оставляем
оригинал, лучше англо чем пусто.

Работает для всех категорий с автозаполнением: Web, YouTube,
text-first (skills/prompts/ideas/portfolio/misc), Designs.
